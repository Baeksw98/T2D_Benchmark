# Public Data Card

This repository ships three kinds of data. None of it contains real patient
data, gold-rubric content, production prompts, or raw model outputs.

## 1. Synthetic vignettes (`data/vignettes/`)

- 480 fully synthetic English T2D vignettes (`T2D_V001.json` … `T2D_V480.json`)
  plus `factorial_index.csv`.
- Generated by a deterministic, seeded generator over a complete six-dimension
  factorial grid (CKD × HF × ASCVD × MASLD × BMI × HbA1c = 480).
- Each record is a longitudinal outpatient narrative with the factorial cell and
  complexity/trap tags attached. Schema: [`data/vignettes/README.md`](data/vignettes/README.md).
- Not sampled from, or linked to, any real patient. Reviewed for clinical
  plausibility per [`VALIDATION.md`](VALIDATION.md).

## 2. Released G-theory estimates (`data/g_theory/`)

- `variance_components.csv` — per-question variance decomposition (15 reliability
  questions).
- `coefficients.csv` — dependability/generalizability coefficients and errors of
  measurement (per configuration × reasoning part).
- These are **aggregate** estimates derived from the withheld raw rater scores.
  They contain no per-patient scores, rater identities, or rubric text. The
  arithmetic linking them to the reported Φ / SEM / SDD is in
  [`VARIANCE_COMPONENTS.md`](VARIANCE_COMPONENTS.md).

## 3. Synthetic demo data (`data/demo/`)

- `synthetic_score_matrix.csv` — a balanced, fully crossed `object × prompt ×
  occasion` matrix (540 rows) for the G-theory runner.
- `synthetic_protocol_scores.csv` — a Baseline vs DRG score table (1,800 rows)
  for the KS / paired-difference analysis.
- Both are deterministic (SHA256-seeded) artificial tables. They are shaped only
  to exercise the pipeline end-to-end; **the numbers they produce are not the
  paper's results.** Regenerate with
  `python -m t2d_benchmark.analysis.generate_synthetic_demo_data --target both`.

## Privacy and release boundary

The release excludes, by design: patient-specific gold-standard rubric content,
production CDSS prompt templates (which carry safety-guard instructions), raw AI
rater outputs, and internal serving/deployment configuration. The automated
release audit (`python -m t2d_benchmark.audit_release --root .`) checks that none
of these markers appear in the released tree.
